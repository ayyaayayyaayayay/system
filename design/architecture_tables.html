Current System Architecture (Implemented)

Layer Component Technology / Approach
Data Protection Password Storage Plaintext password storage and direct password comparison in login flow (no password hashing yet).
Authentication Login and Session Entry PHP API username/password authentication plus role shortcut login and client-side fallback session handling.
Access Control Role-Based Authorization Role checks at app/API level for dean/professor ownership and action permissions on faculty paper workflows.
Reporting Faculty Acknowledgement Reports FPDF + FPDI-based CHED-style PDF rendering from template with overlay fields and generated file output.
Communication Frontend-Backend API Calls JSON-based HTTP fetch requests from panel JavaScript modules to PHP endpoints.
Resilience Backup and Recovery Backup actions currently UI placeholders only (no actual scheduled backup/restore execution pipeline yet).
Network Security Perimeter Security No repository-level Cloudflare/WAF/DDoS implementation in application code (deployment-managed).
Transport Data Transmission Security No application-enforced HTTPS/TLS redirect logic in repo; transport security depends on server configuration.
Application Server-Side Framework Custom PHP endpoint architecture (no full framework such as Laravel/Symfony in this codebase).
Application Client-Side Interface HTML5, CSS3, JavaScript with responsive layout behavior via viewport and media-query rules.
AI and Analytics Comment Summarization and Insights Local rule-based summarization and lexicon frequency counting (no external AI API integration).
Data Management Database System MySQL via PDO with relational schema usage in state helpers and normalized lookup/profile table logic.
